Feedback Page

The Feedback page (feedback.html) was designed to collect user feedback in a structured, accessible way. It uses multiple HTML elements, including:

  • Form Elements: Inputs for name, email, telephone, address, and comments, using appropriate types (text, email, tel).
  • Validation: HTML5 validation with required attributes, input types, and title attributes for custom messages.
  • Character Counter: A character count paragraph for the comments field, dynamically updated by JavaScript (in FeedbackForm.js).
  • Dropdown and Multi-Select: A select menu for feedback topic and a multiple selection box for improvement suggestions.
  • Radio Buttons: To choose if users recommend the website.
  • Rating Input: Emoji radio buttons to rate the experience.
  • File Upload: Accepts images, PDF, and docx files.
  • Accessibility: All inputs have associated labels and placeholders.
  • Styling: The form uses CSS for spacing, colors, and hover states to improve usability.

I ensured that the page is accessible by including alt text, clear labels, and keyboard-friendly navigation.

Team Page

The Team page (team.html) showcases the team members interactively. Its structure includes:

  • Grid Layout: Each member is displayed in a styled box with image and details.
  • Hover Effects: Thumbnails are designed to be visually engaging on hover (this can be further improved with CSS transitions).
  • HTML Semantics: Proper use of headings, paragraphs, and lists to organize information.
  • Accessibility: All images include descriptive alt attributes.
  • Responsiveness: The layout adjusts using CSS Grid or Flexbox (depending on CSS implementation).

The page adheres to the global styling and integrates with the site template for a consistent look.

Accessibility

All my pages include:

  • Descriptive alt text for images.
  • Labels linked to form fields.
  • Consistent heading structure for screen readers.
  • Keyboard navigable interactive elements.

Challenges and Lessons Learned

One challenge was creating the Feedback form with all required input types, validation, and accessible labels. I solved this by carefully planning the HTML structure and testing the form in multiple browsers. Another challenge was aligning the Team page layout consistently across screen sizes, which I addressed with CSS Grid and flexbox.

I learned the importance of semantic HTML, consistent styling, and validating every detail to ensure a professional result.
