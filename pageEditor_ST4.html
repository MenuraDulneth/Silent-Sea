My Role

I was responsible for developing the User Profile, Sitemap and contributing to the Ocean and Human Life content page, both aligned with SDG 14: Life Below Water.

User Profile Page

The User Profile Page presents a detailed and interactive view of the user’s identity and commitment to SDG 14: Life Below Water. It features a profile image, biography, skill bars, and clearly separated sections for goals, frustrations, and experience in marine conservation. The layout is fully responsive using HTML5 and CSS3, with a sidebar and main content area built using Flexbox. An availability calendar helps coordinate volunteer work, while a contact form ensures communication and engagement. Accessibility considerations such as semantic structure and color contrast have been integrated to support a wide range of users.



Accessibility

The User Profile Page is thoughtfully designed with accessibility in mind, ensuring inclusivity for all users. Semantic HTML tags are used to provide a logical reading order and structure. All images include descriptive alt attributes, and color contrast meets WCAG AA standards for readability. The layout supports keyboard-only navigation, and form fields are labeled clearly with required field indicators for screen reader users. Responsive design techniques ensure the content is accessible across different screen sizes and devices, enhancing usability for everyone.

Sitemap

The Site Map Page offers a visually engaging and interactive overview of the website's structure, built with semantic HTML5, CSS3, SVG, and accessibility features. It displays the hierarchical layout of all main pages and subpages using color-coded, clickable nodes connected by SVG lines. Designed with keyboard navigation, ARIA roles, and responsive media queries, the sitemap enhances user understanding, navigation, and accessibility while supporting the SDG Goal 14: Life Below Water theme.



Accessibility

The Site Map Page is built with accessibility in mind, featuring keyboard navigation support, ARIA roles for screen readers, and responsive design to ensure all users can easily explore the website structure.

Challenges and Lessons Learned

Ensuring responsive layout across devices required careful use of Flexbox and media queries.
Integrating semantic structure into the Site Map and Profile pages taught the importance of logical HTML hierarchy.
Balancing visual appeal with W3C validation highlighted the value of clean, standards-compliant code.

Compliance

The web pages follow JANET rules, ensure user safety, and comply with accessibility and privacy guidelines. Code validation and semantic clarity are maintained across the project.
